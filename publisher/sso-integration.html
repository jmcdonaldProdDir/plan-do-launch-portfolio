Testing

  1. Create the customer in IDP
  2. Create a STUB https://auth-qa.investorcom.com/partner-auth/client-name/saml/ (call it the partner auth folder)
  3. That URL will provide a metadata file that we can send to the client
  4. The client uses the metadata file to configure their system and register us as a valid connection. Once that is done, they send us their metadata.
  5. We use that metadata to set up our configurations and create a match for testing.
  6. Now we're ready for testing,
    • Loop through testing in QA, and problem resolution until all problems are resolved.
  7. Start the production process below.

Production

  1. Deployment of code change to production (that's the hard coding of the stub)
  2. Create the customer in IDP
  3. Create a STUB https://auth.investorcom.com/partner-auth/client-name/saml/ (call it the partner auth folder and send the file to the client.
  4. The client uses the metadata file to configure their system and register us as a valid connection. Once that is done, they send us their metadata.
  5. We use that metadata to set up our configurations and create a match for testing.
  6. Now we're ready for testing,
    • Loop through testing in QA, and problem resolution until all problems are resolved.
  7. Remove all the customer federated ID override flags.
