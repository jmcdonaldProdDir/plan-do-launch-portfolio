Overview

As part of the integration with InvestorCOM software products and service, InvestorCOM can integrate with the customer’s authentication infrastructure.

By configuring the InvestorCOM Identity Provider (IDP) and the customer’s authentication server to trust each other over OpenID Connect or SAML 2.0 protocols, the InvestorCOM IDP can delegate all authentication to use our customer’s authentication.

The Process

The process for implementation is as follows:

  • Customer Protocol Decision. The customer will decide which protocol they prefer, OpenID Connect or SAML 2.0.
  • Gathering Configuration Details. InvestorCOM will provide configuration details that the customer will use to configure their systems and InvestorCOM will provide a list of details required by the customer.
  • User Acceptance Testing Setup. Both parties will configure their UAT environments before testing.
  • User Acceptance Testing. Technical people from both parties will have a meeting to test the configuration and verify there are no issues with the handshake. If any problems are found, they can be resolved before moving to production testing.
  • Production Testing Setup. Both parties will configure their production environments before testing.
  • Production Testing. Technical people from both parties will have a meeting to test the configuration and verify there are no issues with the handshake.

Supported Flows

InvestorCOM supports Service Provider (SP) Initiated Flow and does not support IDP Initiated Flow.
