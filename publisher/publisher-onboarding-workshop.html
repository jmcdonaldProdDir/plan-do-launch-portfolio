Introducing Publisher

The InvestorCOM Publisher Solution

Use InvestorCOM Publisher to automate the process of composing, translating and publishing regulatory documents such as MRFPs, Financial Statements, Fund Facts, Prospectuses, and any data-intensive regulatory document. From data to SEDAR-ready documents, Publisher delivers on-time, every time.

Digital Publishing vs Typesetting

From data to print-ready PDFs in one platform – that’s the power of Publisher.

Publisher manages the complex process of translating client data to high-quality regulatory documents on a single platform.

Production proofs are ready for online review, edit and approval within hours of importing client data.

Global content changes are made within seconds with new documents ready for review and approval, eliminating traditional delays with document edits. No version control issues. No PDF markups.

Standards (data, document templates, business rules, workflow)

Template standards exist for Financial Statements, MRFP, Fund Facts, QPDs and consolidated books. Documents include style guides, data standards, and data exchange standards.

Onboarding Experience Overview

Phase 1: Assessment (3 weeks)

Fit Analysis

  • Fund and document review
  • High level field mapping
  • Data analysis
  • Gap assessment

Gap Review

  • Review fit
  • Identify gaps

Phase 1b: Assessment - Enhanced Fit Analysis

  • Optional phase determined by the client
  • Goal = tailor the fit analysis to client
  • Use of standard templates with client-specific data
  • Assumptions:
    • InvestorCOM to create static data for up to 5 funds
    • Client to create BAU and Confluence data for 5 funds
    • Publisher will generate sample documents in either a standard 1-column or 2-column variation for Statements of Financial Positions and Statements of Changes in Net Assets sections
    • Standard fields, layouts, and formatting codes will be used
    • No programming changes to adapt layouts, add new fields or adjust formatting will be done during the enhanced fit assessment. These will be logged in the gap assessment and remediation plan

Phase 2: Planning (1 week)

  1. Gap remediation plan
    1. New fields (text, tables, charts)
    2. AODA
    3. Design gaps
    4. Data availability
  2. Organizational commitments (e.g. filing dates, blackout periods)
  3. Roles and responsibility review
  4. Finalize plan
    1. Scope
    2. Cost
    3. Time
    4. Approved SOW

Phase 3: Project Execution (14 weeks)

  1. Finalize Specification
  2. Complete data mapping
  3. Document business rules and create static data
  4. Build
    1. Adjust templates
    2. Create BAU & Confluence data (client)
    3. Adjust data ingestion
  5. Test
    1. QA, User Acceptance Testing and sign off

Phase 4: Production Business-as-Usual

  1. First production cycle
  2. Transition to BAU Support team
  3. Self-serve data feeds and document production
  4. Adjustments as needed
    1. Fund management (removal)
    2. Addition of new data fields
    3. Business rule changes
    4. Design changes
  5. Regular account check-ins

Roles & Responsibilities (RACI Chart)

RACI = (R)esponsible, (A)ccountable, (C)onsult, (I)nform

  1. Responsible Does the work to complete the task.
  2. Accountable Delegates work and is the last one to review the task or deliverable before it's deemed complete.
  3. Consulted Provides input based on how it will impact their project work or their domain of expertise on the deliverable itself.
  4. Informed. Needs to be kept in the loop on project progress, rather than roped into details of every deliverable.
