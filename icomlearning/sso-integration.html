SSO Integration

SSO USING SAML2.0 MASTER CHECKLIST

Testing Environment Setup

  1. Receive client's test metadata file
  2. Create the test customer in IDP (test environment)
  3. Configure our test environment with client's metadata
  4. Create a test STUB https://auth.investorcom.com/partner-auth/client-name/saml/ in QA environment
  5. Send our metadata file to client for their test environment setup
  6. Wait for client to configure their test environment
  7. Perform initial connection testing
  8. Loop through QA testing and resolve any issues
  9. Once testing is successful, proceed to production setup

Production Deployment

  1. Receive client's production metadata file
  2. Create the customer in production IDP
  3. Configure production environment with client's metadata
  4. Deploy code changes to production environment
  5. Create production STUB https://auth.investorcom.com/partner-auth/client-name/saml/
  6. Send our production metadata file to client
  7. Wait for client to configure their production environment
  8. Perform production validation testing
  9. Remove all customer federated ID override flags
  10. Monitor production implementation
