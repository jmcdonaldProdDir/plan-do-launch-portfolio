Federated Identity Integration Configuration - OIDC

Overview

This document lists details required to configure InvestorCOM customer authentication server to be federated into the InvestorCOM authentication.

Context

Context Details
Protocol OpenID Connect (OIDC)
Environment Production

Details Required by Customer

Required Element Details
Sign In callback url https://auth.investorcom.com/partner-auth/customer-code/signin-oidc
Sign Out callback url https://auth.investorcom.com/partner-auth/customer-code/oidc-signout/callback-oidc

Details required by InvestorCOM

Required Configurations

Configuration Details
Client ID Customer configured id the authority server will recognize as InvestorCOM.
Authority URL URL of the customer authority server.
Secret Random value for security connection
Sign out URL (optional) URL to send user to after they log out. Some customers send users to their own portal.

Required Scopes

The following scopes must be supported:

Scope Details
openid To provide basic OIDC information claims
profile To provide basic user information claims
email To provide user email claim

Required Claims

The following claims are required and assumed to match these names. If your claim names differ, please provide their exact names.

Claim Details
email The user’s unique email address
given_name The user’s first name
family_name
 The user’s last name
name The user's display name

Firewall

The following URL will need to be allowed through the firewall.

https://auth.investorcom.com
