    Peer Compare Configurations

    PeerCompare downloads data from sources nightly then updates that data by adding each security to a Peer Group based upon defined matching criteria. Matching criteria are easily changed; however, changes take a few days.

    PeerCompare configurations include:

    1. Search domain
    2. Allow for recommendations of non-product-shelf securities (Restrict PeerCompare to Client Shelf Only)
    3. Prioritize PeerCompare Listing by with Default Manufacturers
    4. Allow Recommendations 
    5. Allow Disclosures
    6. Allow PeerRating in PeerCompare
    7. Restrict Actions to Product Shelf
    8. Send Confirmation Email For Compliance Events
    9. Secondary Compliance Email Target

    PeerCompare and ShelfAnalyzer security type configurations:

    1. Peer Group matching criteria
    2. Security identification criteria
    3. Security measure/compare criteria
    4. Security reveal configuration

    Search Domain Configuration

    The PeerCompare function compares the security to peers on the Product Shelf with the same matching criteria. Uses cases:

    1. Setting the search domain as the Universe of products: 
      Use Case: An Advisor receives a client's request to add a new security to their portfolio. The Advisor's firm does not have the requested security on its Product Shelf. With the Search Domain configured to search the Universe of securities, the Advisor can find the targeted fund and compare it to the Firms' Product Shelf to find reasonably available alternatives.

    2. Setting the search/compare domain as the Product Shelf: 
      Use Case: An Advisor receives a client's request to add a new security to their portfolio. The Advisor's firm does not have the requested security on their Product Shelf. With the Search Domain configured to search the Product Shelf, the Advisor cannot find the targeted fund and compare it to the Firms' Product Shelf to find reasonably available alternatives.

      Recommended Configuration of the Search Domain:
      Option 1: Search domain as the Universe of products, compare them to peers on your product shelf.

    Configuration Options
    Search Domain Set the search/compare domain as the Product Shelf

    Prioritize PeerCompare Listing by with Default Manufacturers

    This feature allows dealers to set a Priority Security Manufacturer by security type and set the default sort on the Peer Group results to show the Priority Manufacturer at the top of the list with an indicator beside the security.

    When a Priority Manufacturer is set, a flag (four-pointed star ✧) is added to the PeerCompare listing results to aid the user in identifying the Priority Manufacturer products and the default Peer Group sorting places the Priority Manufacturers at the top of the list.

    Notes:

    • The Manufacturer must have a security on the product shelf, and the spelling must be exactly as in our data.
    • There is a single manufacturer for each security type.
    • We have used the term "manufacturer" as securities use a variety of descriptors to identify the provider, issuer, asset manager or fund company.

    Security Type

    The Security Type in the Configuration Guide is a mandatory matching field. Security types are limited to Mutual Fund, ETF, Seg Fund, Hedge Fund, Pooled Fund, Labour Sponsored Fund, North American Equity, Liquid Alternative, Closed-End Fund, and Structured Note. 

    Security type defines the larger categories like Mutual Fund, ETF, Seg Fund and Equity. We have a further breakdown we recommend, called Instrument Type. Instrument Type breaks the Equities Security Type down into more detailed instruments for comparing Equities. See the definition of Instrument Type below, which is based on the first two letters of the ISO CFI code.

    Instrument Type – Based on first two letters of ISO CFI code

    • CB = REIT
    • CE = ETF
    • CI - Std Fund
    • CM - Collective Investment
    • ED = Depository Receipt
    • EL = Limited Partnership Units
    • EM = Other Equities
    • EP = Preferred Shares
    • ES = Common Shares
    • MC = Other

    Matching Fields

    Use matching fields to build PeerCompare Peer Group "matching criteria." Used in PeerCompare and ShelfAnalyzer, and for the Analytics Engine, the matching fields define the criteria for Peer Group assignments.

    When configuring, use the default fields for Matching configuration or choose any combination up to six fields from Appendix A, Categories: Identification Fields, Basic Information, Fees, Suitability, and ETF Data.

    Identification Fields

    Use identification fields in PeerCompare to display data important to PeerCompare users.

    When configuring, use the default fields for Identification configuration or choose any combination of five fields from Appendix A, Categories: Identification Fields, Basic Information, Sales, Fees, Suitability, ETF Data, and Performance

    Measure/Compare fields

    Measure fields to provide users with comparative information.

    When configuring, use default fields for Identification configuration or choose any combination of five fields from Appendix A, Performance category.

    NOTE: Use ETF data only for ETF Matching and Identification configurations.

    Reveal Fields

    Reveal fields display additional info when the user hovers over the security, e.g., Investment Strategy, Investment Objective.

    Compliance

    There are two PeerCompare components to Compliance, 1. Recommendations – Reasons Why and 2. Disclosures.

    1. A Recommendation captures the Advisors CFR/RegBI obligations and stores the detail in ComplianceExpress for reference and reporting.
    2. A Disclosure captures the Advisor's CFR obligations, includes it in an investor email, and stores it in ComplianceExpress.

    Recommendation Restrictions

    There are two configuration options for this setting:

    Configuration Option
    Restrict recommendations to products on the shelf
    If set to "true," allow recommendations and disclosures to products ONLY on the shelf. 
    		 TRUE
    Restrict recommendations to products on the shelf

    IMPORTANT:

    1. If the configuration option for recommendations is set to false, the search domain should be set to the Universe of Securities.
