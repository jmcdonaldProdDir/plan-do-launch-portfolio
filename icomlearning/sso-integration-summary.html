Single Sign-on Integration Summary

Overview

As part of the integration with InvestorCOM software products and services, InvestorCOM can integrate with the customer’s authentication infrastructure. By configuring the InvestorCOM Identity Provider (IDP) and the customer’s authentication server to trust each other over OpenID Connect or SAML 2.0 protocols, the InvestorCOM IDP can delegate all authentication to the customer’s authentication.

Supported Flows:
InvestorCOM supports the Service Provider (SP) Initiated Flow and does not support the IDP Initiated Flow.

Integration Process

  1. Protocol Decision
    The customer’s IT/Admin team decides on the preferred protocol: OpenID Connect or SAML 2.0.
  2. Configuration Details Exchange
    • Customer provides InvestorCOM with their configuration information (e.g., identifiers, domain information, selected protocol details).
    • InvestorCOM then provides the customer with the necessary URL and additional configuration details tailored to the customer's provided setup.
  3. User Acceptance Testing (UAT) Phase
    • Setup: Both InvestorCOM and the customer configure their UAT environments.
    • Testing: Teams from both parties meet to test the setup and verify functionality. Any issues are resolved here.
  4. Production Phase
    • Setup: Both InvestorCOM and the customer configure their production environments.
    • Testing: A final meeting is held with technical teams from both parties to test the production setup and ensure functionality is as expected.
