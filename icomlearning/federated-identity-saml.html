Federated Identity Integration Configuration - SAML

Overview

This document lists details required to configure InvestorCOM customer authentication server to be federated into the InvestorCOM authentication. InvestorCOM supports Service Provider (SP) Initiated Flow and does not support IDP Initiated Flow.

Context

Context Details
Protocol SAML 2.0
Environment Production

Details Required by Customer

Required Element Details
Sign In callback url https://auth.investorcom.com/partner-auth/customer-code/saml

Details required by InvestorCOM

Required Configurations

Send us your metadata file and, your email address pattern.

 SAML IDP URL

 Signing key used in handshake
IDP Entity ID
 Entity ID for the SAML IDP
Email address pattern
 The pattern to recognize an incoming user’s email address. This is the customer domain; i.e. @InvestorCOMcorp.com

Required Claims

The following claims are required and assumed to match these names. If your claim names differ, please provide their exact names.

Claim Details
email The user’s unique email address
given_name The user’s first name
family_name
 The user’s last name
name The user's display name

Firewall

The following URL will need to be allowed through the firewall.

https://auth.investorcom.com

Supported Flows

InvestorCOM supports Service Provider (SP) Initiated Flow and does not support IDP Initiated Flow.
